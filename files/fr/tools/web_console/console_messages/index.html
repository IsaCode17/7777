Chaque message est affiché sur une nouvelle ligne.

Nom du fichier et numéro de ligne

Pour JavaScript, CSS et les messages de l'API console, le message peut être associé à une ligne de code spécifique. La console fournit alors un lien vers le nom du fichier et le numéro de ligne qui a généré le message.

Depuis Firefox 36, cela inclut le nombre de colones également.

Par défaut, la console est effacée à chaque fois que vous naviguez sur une nouvelle page ou que vous rechargez la page courante. Pour modifier ce comportement, cochez "Activer les journaux persistants" dans les paramètres de la Boite à outils.

Categories de messages

Requêtes réseau

Les messages réseau ne sont pas affichés par défaut. Il faut utiliser la fonctionnalité de filtrage pour les afficher.

Les requêtes résseau sont affichées sous cette forme :

Catégorie Indique que ce message concerne une requête HTTP
Méthode

La méthode HTTP utilisée

Si la requête à été faite avec XMLHttpRequest, il y a une notre aditionelle.
URI L'URI cible
Résumé La version HTTP, le code de statut, et le temps qu'il a fallu pour la terminer

Si vous cliquez sur le message, vous verrez une fenêtre contenant plus de détails sur la requête et la réponse :

Défiler vers le bas révèle les en-têtes de réponses. Par défaut, la Console Web n'enregistre pas les contenus de la requête et la réponse : pour le faire, activez le menu contextuel de la Console Web et choisissez "Journaliser le contenu des requêtes et des réponses". Rechargez la page, et vous les verrez dans cette fenêtre d'inspection.

Seul le premier méga-octet de données est enregistré pour le contenu de chaque requête ou réponse. Les requêtes et les réponses très lourdes seront tronquées.

Les logs de messages réseaux ne sont pas montrés par defaut. Utiliser la fonction filtre pour les afficher.

The site uses a certificate whose signature uses the SHA-1 hash algorithm.

SHA-1 is still still widely used in certificates, but it is starting to show its age. Web sites and Certification Authorities are encouraged to switch to stronger hash algorithms in future. See the Weak Signature Algorithm article for more details.

Note that the SHA-1 certificate may not be your site's own certificate, but may be the certificate belonging to a Certification Authority that was used to sign your site's certificate.

Journalisation

Les messages de journalisation des Shared Workers, Service Workers, des modules complémentaires et des Chrome Workers ne sont pas affichés par défaut. Il faut utiliser le filtrage pour les afficher.

La catégorie Journalisation inclut les messages enregistrés avec l'API Console.

La console web prend en charge les messages de l'API Console suivants :

  • assert()
  • clear()
  • count()
  • dir()
  • dirxml()
  • error()
  • exception()
  • group()
  • groupEnd()
  • info()
  • log()
  • table()
  • time()
  • timeEnd()
  • trace()
  • warn()

La console affiche la trace de la pile d'appel pour tous les messages d'erreur :

function toto() {
  console.error("ça explose");
}

function truc() {
  toto();
}

function faireQqc() {
 truc();
}

faireQqc();

Serveur

La journalisation des messages côté serveur a été introduite avec Firefox 43 puis retirée avec Firefox 56. Vous pouvez installer l'extension Chrome Logger pour retrouver cette fonctionnalité.

Avec l'extension Chrome Logger, la console web peut afficher les messages envoyés par le serveur. Cela permet d'utiliser la console web pour déboguer du code côté serveur.

Cette extension utilise le protocole Chrome Logger. Voici un rapide résumé de son fonctionnement :

  • Votre code côté serveur (Python, PHP, Node.js, etc) inclut une bibliothèque qui fournit une API de console.
  • Le code côté serveur utilise cette API pour la journalisation.
  • La bibliothèque côté serveur crée des objets JSON à partir des messages et les encode pour les transmettre.
  • Les messages sont transmis au client sous la forme d'un en-tête de réponse HTTP intitulé X-ChromeLogger-Data.
  • La console décode ces en-têtes et les affiche.

Pour trouver une bibliothèque correspondant côté serveur, voir la documentation de Chrome Logger.

Interpréteur entrée/sortie

Les commandes envoyées au navigateur avec l'interpréteur JavaScript de la console web, et les réponses correspondantes sont affichées parmi les messages de la console.

Pour les réponses qui contiennent des objets ou des variables, les options de menu contextuel suivantes sont disponibles :

Faire apparaitre dans l'inspecteur
Afficher le nœud DOM sélectionner dans l'onglet Inspecteur.
Enregistrer comme variable globale
Créer une variable globale (intitulée temp0, ou temp1, etc.) dont la valeur est celle de l'objet sélectionné. Le nom de la variable apparaît en entrée de l'interpréteur et sa valeur apparaît comme une réponse.
Copier l'objet
Copier l'objet sélectionné dans le presse-papier.

Filtrer et rechercher

Filtrer par catégorie

Vous pouvez utiliser la barre d'outils située en haut afin de limiter les résultats affichés.

Pour voir les messages d'une ou plusieurs catégories précises, on cliquera sur le bouton correspondant à la catégorie voulue (Erreurs, CSS, etc.).

Pour les catégories Erreurs et Avertissements, lorsqu'on les désactive, un nombre apparaît près le texte du bouton pour indiquer le nombre de messages disponibles pour le type correspondant. Par exemple, "Avertissements (25)".

Les requêtes réseaux avec des codes de réponse sur les intervalles 400-499 (erreur du client) et 500-599 (erreur du serveur) sont considérées comme des erreurs. Leur affichage est contrôlé par le bouton Erreurs et non par le bouton Requêtes.

Filtrer par texte

Pour ne voir que les messages qui contiennent une chaîne de caractères donnée, saisissez le texte voulu dans le champ intitulé "Filtrer". Ainsi, si vous entrez le texte img dans ce champ, vous pourrez obtenir une liste comme celle-ci :

Un icône de croix apparaît à l'extrémité droite du champ lorsqu'un texte a été saisi. En cliquant dessus, cela supprime le filtre et affiche la liste non-filtrée.

Vous pouvez exclure un texte choisi en utilisant - comme préfixe. Ainsi, -img affichera uniquement les messages qui ne contiennent pas la chaîne de caractères img.

Filtrer avec des expressions rationnelles

Il est aussi possible d'utiliser une epxression rationnelle afin de filtrer ce qui est affiché dans la console. Dans la vidéo suivante, on voit comment filtrer en utilisant deux expressions rationnelles : /(cool|rad)/ et /(cool)/.

{{EmbedYouTube("E6bGOe2fvW0")}}

Là encore, on peut prendre l'opposé d'une expression rationnelle en la préfixant avec -. Ainsi, -/(cool|rad)/ affichera uniquement les messages qui ne suivent pas l'expression rationnelle /(cool|rad)/.

Purger le journal

Enfin, vous pouvez utiliser l'icône de la poubelle située à gauche pour vider le contenu de la console. Lorsque la console est vidée, le cache de la console est également vidé. Cela empêche la réapparition des messages d'erreur précédemment affichés lors de la réouverture de la console.
