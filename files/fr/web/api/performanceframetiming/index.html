{{SeeCompatTable}}{{APIRef("Frame Timing API")}}

PerformanceFrameTiming est une interface abstraite qui fournit des données de synchronisation « frame » sur la boucle d'événements du navigateur. Une « frame » représente la quantité de travail qu'un navigateur effectue dans une boucle d'événements, comme le traitement des événements DOM, le redimensionnement, le défilement, le rendu, les animations CSS, etc.Un frame rate de 60fps (images par seconde) pour un taux de rafraîchissement de 60Hz est l'objectif pour une bonne expérience utilisateur responsive. Cela signifie que le navigateur doit traiter une image en environ 16,7 ms.

Une application peut enregistrer un {{domxref("PerformanceObserver")}} pour une « frame » de {{domxref("PerformanceEntry","l'entrée de performance","",1)}} et l'observateur peut récupérer des données sur la durée de chaque événement visuel. Ces informations peuvent être utilisées pour aider à identifier les zones qui prennent trop de temps pour offrir une bonne expérience utilisateur.

{{InheritanceDiagram}}

Propriétés

Cette interface n'a pas de propriétés mais elle étend les propriétés de {{domxref("PerformanceEntry")}} suivantes (pour « frame » de {{domxref ("PerformanceEntry.entryType","l'entrée de performance","",1)}}) en qualifiant et en contraignant les propriétés comme suit :

{{domxref("PerformanceEntry.entryType")}}
Retourne « frame ».
{{domxref("PerformanceEntry.name")}}
Renvoie l'adresse du document.
{{domxref("PerformanceEntry.startTime")}}
Retourne le {{domxref("DOMHighResTimeStamp", "timestamp")}} (moment) où le cadre a été lancé.
{{domxref("PerformanceEntry.duration")}}
Retourne un {{domxref("DOMHighResTimeStamp", "timestamp")}} indiquant la différence entre le startTime de deux images successives.

Méthodes

Cette interface n'a pas de méthodes.
