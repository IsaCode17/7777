Les API Permissions fournissent une manière cohérente de demander programatiquement le statut des permissions dans le contexte courrant — par exemple, si l'autorisation d'utiliser l'API est accordée ou refusée, et si l'utilisateur sera invité à indiquer si une autorisation doit être accordée pour une API.

Conceptes et usages

Historiquement les differentes API géraient leur propres permissions de façon peu cohérente — par example l'API Notifications permet de vérifier explicitement sont status d'autorisation et et d'en faire la demande, alors que l'API de Geolocation ne le fait pas (ce qui peut poser des problèmes si l'utilisateur décline la première demande d'autorisation). L'API de permissions fournit l'outil pour autoriser les développeurs à implémenter une meilleure expérience utilisateur.

La propriété permissions a rendu disponible dans l'objet {{domxref("Navigator")}}, ainsi que dans le contexte standard du navigateur et dans le contexte du worker ({{domxref("WorkerNavigator")}} — donc les vérificationd de permissions sont disponible dans les workers), et retournent un objet {{domxref("Permissions")}} qui fournit l'accés aux fonctionalité de l'API de gestion des permissions.

Une fois que vous avez cette objet vous pouvez effectuer une tache nécéssitant des permissions, par exemple demander la permission en utilisant la méthode {{domxref("Permissions.query()")}} pour retourner une promesse qui résous avec {{domxref("PermissionStatus")}} pour une API spécifique.

Tous les status ne sont pas accessibles via l'API. Voici quelques API notables prenant en compte les permissions :

  • API Clipboard
  • API Notifications
  • API Push

  • Web MIDI API

Davantage d'APIs devraient être supportées par l'API Permissions dans le futur.

Interfaces

{{domxref("Navigator.permissions")}} et {{domxref("WorkerNavigator.permissions")}} {{readonlyinline}}
Fournit un accés à l'objet {{domxref("Permissions")}} depuis le contexte principal ainsi que celui du worker.
{{domxref("Permissions")}}
Fournit les principales fonctionalités de l'API Permissions, à savoir les methodes de demande et de révocation des permissions.
{{domxref("PermissionStatus")}}
Fournit l'accés au status actuel d'une permission, et un gestionnaire d'évenements pour répondre aux changements de status d'une permission.
