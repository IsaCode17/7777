{{DefaultAPISidebar("Performance Timeline API")}}

L'API Performance Timeline définit des extensions de l'interface {{domxref("Performance")}} pour prendre en charge les mesures de latence côté client dans les applications. Ces extensions fournissent des interfaces permettant de récupérer les {{domxref("PerformanceEntry", "paramètres de saisie des performances", '', 1)}} en fonction de critères de filtrage spécifiques. La norme comprend également, des interfaces qui permettent à une application de définir des retours (« callbacks ») d'{{anch("Performance_Observers", "observateur des performances", '', 1)}}, qui sont notifiés lorsque des événements de performance spécifiques sont ajoutés à la chronologie des performances du navigateur.

Ce document fournit un aperçu des interfaces de la norme. Pour plus de détails sur les interfaces, voir les pages de référence et Utilisation de Performance Timeline.

{{AvailableInWorkers}}

Extensions de Performance

L'API Performance Timeline étend l'interface {{domxref("Performance")}} avec trois méthodes qui fournissent différents mécanismes pour obtenir un ensemble d'{{domxref("PerformanceEntry", "enregistrements des performances (métriques)", "", 1)}}, en fonction des critères de filtrage spécifiés. Ces méthodes sont les suivantes :

{{domxref("Performance.getEntries","getEntries()")}}
Renvoie toutes les {{domxref("PerformanceEntry", "entrées de performances", "", 1)}} enregistrées ou, éventuellement, les entrées basées sur les {{domxref("PerformanceEntry.name", "name")}}, {{domxref("PerformanceEntry.entryType", "type de performance", "", 1)}} et/ou les {{domxref("PerformanceEntry.initiatorType", "initiatorType")}} spécifiés. (tel qu'un élément HTML).
{{domxref("Performance.getEntriesByName","getEntriesByName()")}}
Renvoie les {{domxref("PerformanceEntry", "entrées de performances", "", 1)}} enregistrées en fonction du {{domxref("PerformanceEntry.name", "name")}} spécifié et éventuellement du {{domxref("PerformanceEntry.entryType", "type de performance", "", 1)}}.
{{domxref("Performance.getEntriesByType","getEntriesByType()")}}
Renvoie les {{domxref("PerformanceEntry", "entrées de perfornances", "", 1)}} enregistrées en fonction des {{domxref("PerformanceEntry.entryType", "types de performances", "", 1)}} spécifiés.

L'interface PerformanceEntry

The {{domxref("PerformanceEntry")}} interface encapsulates a single performance entry — that is, a single data point or metric in the performance timeline. This interface has the following four properties, and these properties are extended (with additional constraints) by other interfaces (such as {{domxref("PerformanceMark")}}):

{{domxref("PerformanceEntry.name","name")}}
Le nom de l'entrée de performance lorsque la métrique a été créée.
{{domxref("PerformanceEntry.entryType","entryType")}}
Le type de mesure de performance (par exemple, « mark »).
{{domxref("PerformanceEntry.startTime","startTime")}}
Un {{domxref("DOMHighResTimeStamp", "horodatage haute résolution", "", 1)}} représentant l'heure de départ de l'entrée de performance.
{{domxref("PerformanceEntry.duration","duration")}}
Un {{domxref("DOMHighResTimeStamp", "horodatage haute résolution", "", 1)}} représentant la valeur temporelle de la durée de l'événement de performance. (Certains {{domxref("PerformanceEntry.entryType", "types d'entrée", "", 1)}} de performance n'ont pas de concept de durée et cette valeur est fixée à '0' pour ces types).

Cette interface comprend une méthode {{domxref("PerformanceEntry.toJSON", "toJSON()")}} qui renvoie la sérialisation de l'objet {{domxref("PerformanceEntry")}}. La sérialisation est spécifique à l'entrée {{domxref("PerformanceEntry.entryType", "type de performance", "", 1)}}.

Observateurs des performances

{{SeeCompatTable}}

Les interfaces d'observateurs de performance permettent à une application d'enregistrer un observateur pour des types d'événements de performance spécifiques, et lorsqu'un de ces types d'événements est enregistré, l'application est notifiée de l'événement via la fonction de rappel de l'observateur qui a été spécifiée lors de la création de l'observateur.

Lorsque l'observateur (rappel ou « callback ») est invoqué, les paramètres du rappel incluent une {{domxref("PerformanceObserverEntryList", "liste des entrées de l'observateur de performance", "", 1)}} qui contient uniquement des {{domxref("PerformanceEntry", "entrées de performance", "", 1)}} observées. C'est-à-dire que la liste contient uniquement des entrées pour les types d'événements qui ont été spécifiés lorsque la méthode {{domxref("PerformanceObserver.observe", "observe()")}} de l'observateur a été invoquée. L'interface {{domxref("PerformanceObserverEntryList")}} possède les trois mêmes méthodes getEntries*() que l'interface {{domxref("Performance")}}. Cependant, notez qu'il y a une différence clé avec ces méthodes ; les versions {{domxref("PerformanceObserverEntryList")}} sont utilisées pour récupérer les entrées de performance observées dans le rappel de l'observateur.

Outre la méthode {{domxref("PerformanceObserver.observe", "observe()")}} de l'interface {{domxref("PerformanceObserver")}} (qui sert à enregistrer les {{domxref("PerformanceEntry.entryType", "types d'entrées", "", 1)}} pour observer), l'interface {{domxref("PerformanceObserver")}} possède également une méthode {{domxref("PerformanceObserver.disconnect", "disconnect()")}} qui empêche un observateur de recevoir d'autres événements.

Les observateurs de performance ont été ajoutés au Niveau 2 de la norme et n'ont pas été implémentés à grande échelle.

État de la mise en œuvre

Un résumé de l'état de mise en œuvre des interfaces est fourni ci-dessous, avec un lien vers des informations plus détaillées.

  • Les extensions de l'interface Performance : Comme le montre le tableau de Compatibilité des navigateurs de l'interface {{domxref("Performance")}}, la plupart de ces interfaces sont largement prises en charge par les navigateurs de bureau et bénéficient d'un support moindre sur les appareils mobiles.
  • PerformanceEntry : Comme le montre le tableau de Compatibilité des navigateurs de l'interface {{domxref("PerformanceEntry")}}, la plupart de ces interfaces sont largement prises en charge par les navigateurs de bureau et le sont moins sur les appareils mobiles.
  • Observateur de performance {{experimental_inline}} : Comme le montre le tableau de Compatibilité des navigateurs de l'interface {{domxref("PerformanceObserver")}}, cette interface n'est pas implémentée sur les navigateurs.

Pour tester la prise en charge de ces interfaces par votre navigateur, exécutez l'application perf-api-support.
