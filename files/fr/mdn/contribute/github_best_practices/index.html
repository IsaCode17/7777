Cette page contient les meilleures pratiques pour travailler avec GitHub pour contribuer au MDN, principalement axées sur la façon de travailler avec les problèmes.

Lorsque vous choisissez un problème GitHub sur lequel travailler

  1. Écrivez un commentaire dans le ticket en disant que vous aimeriez vous en occuper, et nous vous y affecterons.
    • Si quelqu'un d'autre est déjà affecté au problème :
      1. Si cela s'est passé il y a plus d'une semaine et qu'il n'y a pas eu beaucoup d'activité, mentionnez-les personnes et demandez-leur si vous pouvez reprendre le ticket ou les aider à terminer le projet.
        • S'ils sont d'accord pour que vous le preniez, nous vous y affecterons et nous les retirerons.
        • S'ils sont d'accord pour que vous le preniez et qu'une partie du travail a déjà été effectuée, ou si l'accord prévoit que vous les aidiez, nous vous y affecterons à leurs côtés.
      2. Si c'était il y a moins d'une semaine, soyez patient et donnez-leur une chance d'y travailler.
  2. Si le problème a été marqué comme complet mais nécessitant une révision, et que vous souhaitez le relire, mentionnez la personne (@pseudo) dans les commentaires et dites que vous allez relire sses modifications.

Lorsque vous avez été assigné à un problème

  1. Déterminez le reste des travaux à effectuer.
    • Si le problème est bien décrit et que le travail est assez évident, foncez et faites-le.
    • Si le problème n'est pas bien décrit, et/ou si vous n'êtes pas sûr de ce qui est nécessaire, n'hésitez pas à @mentionner l'auteur et à demander plus d'informations.
    • Si vous ne savez toujours pas à qui vous adresser, demandez de l'aide dans le salon de discussion MDN Web Docs FR sur Matrix.
  2. Une fois que vous pensez avoir résolu un problème, demandez une évaluation dans les commentaires.
  3. Une fois qu'un problème a été révisé avec succès et que les commentaires ont reçu une réponse positive, vous pouvez le marquer comme clos.
  4. Si vous n'avez plus le temps de travailler sur un problème, faites-le nous savoir dans un commentaire afin que nous puissions l'attribuer à quelqu'un d'autre.
