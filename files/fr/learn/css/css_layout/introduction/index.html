Dans cet article, nous reviendrons sur certaines fonctionnalités CSS liées à la disposition du contenu que nous avons déjà abordées dans les modules précédents (par exemple les différentes valeurs pour la propriété display) et nous introduirons certains nouveaux concepts.

Prérequis : Des bases de HTML (voir Introduction à HTML), et une idée du fonctionnement de CSS (voir Premiers pas en CSS).
Objectif : Vous fournir un aperçu des différentes techniques CSS pour organiser, disposer le contenu sur une page. Chaque technique sera approfondie dans le détail dans les tutoriels qui suivent.

Les techniques de disposition CSS permettent de contrôler la façon dont les différents éléments d'une page web sont positionnés par rapport à leur position par défaut dans le « flux normal » de la page, par rapport aux autres éléments alentour, par rapport à l'élément parent qui les contient ou par rapport à la fenêtre/zone d'affichage du navigateur. Les techniques qui seront abordées dans cet article et approfondies dans les tutoriels suivants sont :

  • Le flux normal
  • La propriété display
  • Les boîtes flexibles (flexbox)
  • Les grilles
  • Le flottement des éléments
  • Le positionnement
  • La disposition des tableaux
  • La disposition en colonnes

Chaque technique a ses avantages, inconvénients et cas d'usage. Aucune technique n'est destinée à être utilisée seule. En comprenant le rôle que peut jouer chaque méthode, vous serez en meilleure position pour choisir l'outil le plus adapté aux besoins que vous aurez.

Le flux normal

Le flux normal correspond au comportement par défaut du navigateur quand il agence les pages HTML et que rien n'est fait pour modifier la disposition. Prenons un rapide exemple en HTML :

<p>J'adore mon chat.</p>

<ul>
  <li>Acheter des croquettes</li>
  <li>Faire un peu de sport</li>
  <li>Soutenir les copines et les copains</li>
</ul>

<p>Fin !</p>

On voit ici que le HTML est affiché selon l'ordre dans lequel les éléments apparaissent dans le code source, les éléments sont empilés les uns au dessus des autres : le premier paragraphe est suivi de la liste, elle-même suivie du deuxième paragraphe.

Les éléments qui apparaissent les uns en dessous des autres sont appelés des éléments « de bloc » (block elements en anglais). Ils sont à comparer aux éléments « en ligne » ou en incise (inline elements en anglais) qui apparaissent les uns à la suite des autres comme les mots au sein d'un paragraphe.

Note : la direction selon laquelle les éléments de bloc se suivent est décrite comme « la direction de bloc ». Pour les langues écrites horizontalement (comme le français), la direction de bloc est verticale. Pour les langues écrites verticalement (comme le japonais), cette direction est horizontale. De même, il existe la « direction en ligne » qui décrit la direction selon laquelle le contenu en ligne s'organise.

Pour la plupart des éléments de la page, le flux normal permettra d'obtenir la disposition attendue. Toutefois, pour certaines dispositions plus complexe, il faudra modifier le comportement par défaut grâce aux outils fournis par CSS. Il est toujours important de démarrer avec un document HTML bien structuré car on peut alors s'appuyer sur la disposition par défaut plutôt que d'avoir à lutter contre elle.

Les méthodes qui modifient l'organisation des éléments en CSS sont :

  • La propriété display : les valeurs standards comme block, inline ou inline-block peuvent modifier le comportement des éléments dans le flux normal (un élément de bloc pourra alors se comporter comme un élément en ligne) (voir l'article Types de boîtes CSS pour plus d'informations). Certaines valeurs spécifiques de display permettent d'utiliser des méthodes de disposition à part entière comme les grilles CSS et les boîtes flexibles (flexbox), qui permmettent d'altérer l'organisation des éléments à l'intérieur de leur parent.
  • Le flottement : utiliser la propriété float avec des valeurs comme left permet aux éléments de bloc de se déplacer sur le côté d'un élément, à l'instar des images d'un magazine où le texte suivrait les contours de l'image.
  • La propriété position : celle-ci permet de contrôler finement le placement des boîtes au sein d'autres boîtes. Le positionnement statique (static) correspond au comportement par défaut pour le flux normal mais on peut choisir d'autres valeurs : pour avoir, par exemple, le contenu toujours fixé en haut de la fenêtre du navigateur.
  • La disposition en tableau : les fonctionnalités destinées à la mise en forme des tableaux HTML peuvent être utilisées sur des éléments qui ne sont pas des tableaux grâce à display: table et aux propriétés associées.
  • La disposition en colonnes : les propriétés pour la disposition en colonnes permettent d'organiser le contenu d'un bloc en colonnes, à l'instar de la mise en page des journaux.

La propriété display

Les principales méthodes permettant d'organiser la disposition d'une page en CSS sont obtenues grâce aux valeurs de la propriété display. Cette propriété permet de changer la façon dont les objets s'affichent par défaut. Dans le flux normal, chaque élément possède une valeur par défaut pour la propriété display qui décrit comment ils se comportent. Ainsi, les paragraphes écrits en français s'affichent les uns en dessous des autres car leur mise en forme par défaut utilise display: block. Si vous créez un lien contenant du texte dans un paragraphe, ce lien reste intégré dans le reste du texte et ne passe pas sur une nouvelle ligne. Cela est dû au fait que l'élément <a> se comporte avec display: inline par défaut.

Il est possible de changer cet affichage par défaut. Ainsi, l'élément <li> utilise display: block par défaut et les éléments d'une liste s'affiche donc les uns en dessous des autres. Si on change la valeur de cette propriété pour ces éléments avec inline, ils s'afficheront alors les uns après les autres tels des mots dans une phrase. En fait, on peut changer la valeur de la propriété display pour n'importe quel élément HTML. Cela signifie qu'on peut choisir les éléments HTML en fonction de leur sémantique (leur signification logique) sans se soucier de leur apparence qu'on pourra changer en CSS si besoin.

En plus de pouvoir modifier la présentation par défaut d'un élément et passer de block à inline et vice versa, il existe des méthodes d'organisation plus larges qui utilisent aussi display. Toutefois, avec ces méthodes, il faudra généralement utiliser d'autres propriétés complémentaires. Les deux valeurs que nous allons voir ici sont display: flex et display: grid.

Flexbox

Flexbox (ou « boîtes flexibles » en français) est l'abbréviation utilisée pour le module de spécification CSS Flexible Box Layout. Il s'agit ici de simplifier l'organisation d'objets sur une dimension, que ce soit une ligne ou une colonne. Pour utiliser les boîtes flexibles, on applique display: flex sur l'élément parent des éléments qu'on souhaite disposer. Tous les éléments enfants deviennent alors des objets flexibles. Voyons ce que cela donne dans un exemple simple.

Le balisage HTML qui suit fournit un élément englobant avec une classe wrapper. Dans cet élément, on trouve trois éléments <div>. Par défaut, ces éléments s'affichent comme des éléments de blocs, les uns en dessous des autres.

Maintenant, si on ajoute display: flex à l'élément parent, les trois éléments s'arrangent en colonnes. En effet, ce sont maintenant des objets flexibles et leur affichage est modifié par certaines valeurs initiales fixées sur le conteneur flexible. Ils s'affichent sur une ligne car la valeur initiale de flex-direction qui est associée à l'élément parent est row. Les éléments s'étirent tous à la hauteur de l'élément le plus haut car la valeur initiale de la propriété align-items pour le parent est stretch. Cela signifie que les éléments s'étirent sur la hauteur du conteneur flexible, or celle-ci est définie par l'objet le plus haut. On voit aussi que les éléments sont regroupés au début du conteneur et que l'espace libre est laissé à la fin de la ligne.

Flexbox - Exemple n°1
* {box-sizing: border-box;}

.wrapper > div {
    border-radius: 5px;
    background-color: rgb(207,232,220);
    padding: 1em;
}
    
.wrapper {
  display: flex;
}

<div class="wrapper">
  <div class="box1">Un</div>
  <div class="box2">Deux</div>
  <div class="box3">Trois</div>
</div>

En plus des propriétés précédentes qui s'appliquent au conteneur flexible, on a également des propriétés qui s'appliquent aux objets flexibles. Ces propriétés permettent, entre autres choses, de modifier la flexibilité des éléments, leur permettant de s'étendre ou de se comprimer en fonction de l'espace disponible.

Prenons un exemple rapide où on ajoute la propriété flex à chaque élément enfant avec une valeur de 1. Cela entraînera l'expansion des éléments pour remplir le conteneur (plutôt que de laisser l'espace à la fin). S'il y a plus d'espace les éléments s'élargissent et s'il y a moins d'espace, ils se compressent. Si on ajoute un autre élément au balisage, les éléments rétréciront afin de laisse de l'espace pour le nouvel élément : leur taille s'ajuste selon l'espace du conteneur.

Flexbox - Exemple n°2
    * {box-sizing: border-box;}

    .wrapper > div {
        border-radius: 5px;
        background-color: rgb(207,232,220);
        padding: 1em;
    }
    
.wrapper {
    display: flex;
}

.wrapper > div {
    flex: 1;
}

<div class="wrapper">
    <div class="box1">Un</div>
    <div class="box2">Deux</div>
    <div class="box3">Trois</div>
</div>

La disposition en grille

Si les boîtes flexibles sont conçues pour les dispositions sur une dimension, les grilles CSS permettent d'organiser le contenu selon deux dimensions et d'aligner des objets sur des lignes et des colonnes.

Là encore, on peut utiliser ce mode grâce à une valeur spécifique de la propriété display : display: grid. Dans l'exemple qui suit, on utilise une structure HTML similaire à celles des exemples précédents où on a un conteneur et des éléments enfants. En plus d'utiliser display: grid, on définit aussi certaines pistes pour les lignes et les colonnes sur l'élément parent grâce aux propriétés grid-template-rows et grid-template-columns. On définit trois colonnes de 1fr chacune et deux lignes de 100px. Aucune règle CSS n'est ajoutée aux éléments enfants, ceux-ci sont automatiquement placés sur les cellules de la grille ainsi créée.

Grille - Exemple n°1
    * {box-sizing: border-box;}

    .wrapper > div {
        border-radius: 5px;
        background-color: rgb(207,232,220);
        padding: 1em;
    }
    
.wrapper {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    grid-template-rows: 100px 100px;
    grid-gap: 10px;
}

<div class="wrapper">
    <div class="box1">Un</div>
    <div class="box2">Deux</div>
    <div class="box3">Trois</div>
    <div class="box4">Quatre</div>
    <div class="box5">Cinq</div>
    <div class="box6">Six</div>
</div>

{{ EmbedLiveSample('Grid_1', '300', '330') }}

Une fois qu'on a une grille, on peut explicitement placer les éléments sur celle-ci plutôt que de suivre le placement automatique (qu'on a pu observer dans l'exemple qui précède). Dans le prochain exemple, on définit la même grille mais on utilise uniquement trois éléments enfants. On définit le début et la fin de la ligne pour chaque élément grâce aux propriétés grid-column et grid-row. De cette façon, les éléments occupent plusieurs pistes.

Grille - Exemple n°2
    * {box-sizing: border-box;}

    .wrapper > div {
        border-radius: 5px;
        background-color: rgb(207,232,220);
        padding: 1em;
    }
    
.wrapper {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    grid-template-rows: 100px 100px;
    grid-gap: 10px;
}

.box1 {
    grid-column: 2 / 4;
    grid-row: 1;
}

.box2 {
    grid-column: 1;
    grid-row: 1 / 3;
}

.box3 {
    grid-row: 2;
    grid-column: 3;
}

<div class="wrapper">
    <div class="box1">Un</div>
    <div class="box2">Deux</div>
    <div class="box3">Trois</div>
</div>

Dans la suite de ce guide, nous allons voir d'autres méthodes pour organiser la disposition du contenu. Ces méthodes sont moins importantes pour l'organisation principale d'une page mais permettent de réaliser certaines tâches spécifiques. En comprenant ces différents outils, vous serez en mesure de choisir le type de disposition le plus adapté au composant que vous voudrez réaliser.

Les éléments flottants (floats)

Faire flotter un élément consiste à le retirer du flux normal et à le déplacer à gauche ou à droite. Les éléments de bloc environnants encadrent l'élément autour de sa nouvelle position.

La propriété float possède quatre valeurs possibles :

  • left : l'élément flotte à gauche.
  • right : l'élément flotte à droite.
  • none : aucun flottement n'est appliqué. C'est la valeur par défaut.
  • inherit : indique que la valeur de la propriété float doit être héritée de celle de l'élément parent.

Dans l'exemple qui suit, on fait flotter un élément <div> à gauche et on lui ajoute une marge (via margin) à droite afin de repousser le texte de ce côté de l'élément. L'effet qui en résulte est du texte autour de cette boîte.

Éléments flottants - Exemple
body {
    width: 90%;
    max-width: 900px;
    margin: 0 auto;
}

p {
    line-height: 2;
    word-spacing: 0.1rem;
}

.box {
    background-color: rgb(207,232,220);
    border: 2px solid rgb(79,185,227);
    padding: 10px;
    border-radius: 5px;
}

Les techniques de positionnement

Le positionnement permet de déplacer un élément depuis son emplacement du flux normal vers une autre position. Le positionnement n'est pas à proprement parler une méthode pour créer des dispositions de page, il s'agit plutôt de gérer finement la position de certains objets spécifiques sur la page.

Ceci étant écrit, il existe certaines techniques de disposition qui reposent également sur l'utilisation de la propriété position. Comprendre le positionnement aide également à comprendre le flux normal et l'impact que peut avoir le retrait d'un élément de ce flux.

Il existe cinq types de positionnement à connaître :

  • Le positionnement statique est le positionnement par défaut utilisé par chaque élément. Cela signifie que l'élément occupe sa position normale dans le flux du document. Il n'y a rien de spécial.
  • Le positionnement relatif permet de modifier la position d'un élément sur la page et de le déplacer relativement à sa position dans le flux normal (ce qui permet également de le superposer sur d'autres éléments de la page).
  • Le positionnement absolu déplace un élément hors du flux, comme si celui-ci évoluait sur une couche séparée. La position de l'élément est ensuite déterminée relativement aux bords de la page (matérialisée par l'élément <html>) ou à l'ancêtre le plus proche qui est positionné. Ce positionnement permet d'obtenir des effets complexes comme des onglets où le contenu de chaque onglet s'affiche sur les onglets inactifs ou pour des panneaux d'information qui sont affichés en dehors de l'écran par défaut et qu'on peut faire apparaître via des boutons.
  • Le positionnement fixe est très proche du positionnement absolu mais la position de l'élément est relative à la zone d'affichage du navigateur (viewport) plutôt qu'à un autre élément. Cela permet d'obtenir des éléments toujours visibles à l'écran tandis qu'on fait défiler le reste du contenu.
  • Le positionnement « adhérent » est une méthode plus récente qui permet d'avoir un élément dont la position se comporte comme position: static jusqu'à ce qu'un certain décalage par rapport à la zone d'affichage soit atteint, l'élément se comportera alors comme s'il avait position: fixed.

Exemple de positionnement simple

Pour illustrer ce fonctionnement, nous allons présenter quelques exemples. Ces exemples utilisent la même structure HTML :

}

Le positionnement relatif

Le positionnement relatif permet de décaler la position d'un élément par rapport à celle occupée par défaut dans le flux normal. On peut par exemple utiliser ceci pour déplacer une icône vers le bas afin de l'aligner avec le libellé qui lui correspond. Pour obtenir cet effet, on ajoute la règle CSS suivante :

.positioned {
  position: relative;
  top: 30px;
  left: 30px;
}

Pour le paragraphe du milieu, on utilise la propriété position avec la valeur relative. Toute seule, cette modification n'a aucun effet. On ajoute donc les propriétés top et left. Celles-ci permettent de déplacer l'élément ciblé vers le bas et sur la droite. On pourrait penser que c'est contre-intuitif mais il faut voir que l'élément est poussé depuis les côtés haut et gauche, il se déplace donc vers la droite et vers le bas.

En ajoutant cette règle CSS, on obtient le résultat suivant :

<h1>Positionnement relatif</h1>

<p>Je suis un élément de bloc basique.</p>
<p class="positioned">Voici l'élément positionné de façon relative.</p>
<p>Je suis un élément de bloc basique.</p>
body {
  width: 500px;
  margin: 0 auto;
}

p {
    background-color: rgb(207,232,220);
    border: 2px solid rgb(79,185,227);
    padding: 10px;
    margin: 10px;
    border-radius: 5px;
}

.positioned {
  position: relative;
  background: rgba(255,84,104,.3);
  border: 2px solid rgb(255,84,104);
  top: 30px;
  left: 30px;
}

Le positionnement absolu

Le positionnement absolu permet de complètement retirer un élément du flux normal. L'élément est placé en utilisant des décalages depuis les bords du bloc qui le contient.

Dans l'exemple, nous allons ajouter la règle CSS qui suit :

.positioned {
  position: absolute;
  top: 30px;
  left: 30px;
}

Ici, pour le paragraphe du milieu, on utilise la propriété position avec la valeur absolute. On utilise les mếmes propriétés top et left qu'avant. Voici le résultat qu'on obtient :

<p>Je suis un élément de bloc basique.</p>

  margin: 0 auto;
}

    border: 2px solid rgb(79,185,227);
    padding: 10px;
    margin: 10px;
    border-radius: 5px;
}

.positioned {
    position: absolute;
    background: rgba(255,84,104,.3);

La différence est frappante. L'élément positionné est désormais complètement séparé du reste de la disposition. Il est affiché par-dessus le reste. Les deux autres paragraphes sont désormais rapprochés, comme si leur voisin n'existait pas. Les propriétés top et left ont un effet différent que pour le positionnement relatif. Ici, les décalages sont calculés à partir du haut et de la gauche de la page. Il est possible d'utiliser un élément parent plutôt que la page pour ce décalage, nous verrons comment faire dans l'article dédié au positionnement.

Le positionnement fixe

Le positionnement fixe retire les éléments du flux comme le positionnement absolu. Toutefois, les décalages sont appliqués par rapport à la zone d'affichage (viewport) plutôt que sur le conteneur. L'objet ciblé reste alors fixe dans la zone d'affichage et on peut créer des effets comme un menu qui reste affiché au même endroit quel que soit le défilement appliqué à la page.

Pour cet exemple, on utilisera un fragment HTML avec trois paragraphes de texte afin de pouvoir faire défiler le contenu et une boîte aura position: fixed.

<h1>Positionnement fixe</h1>

<div class="positioned">Fixe</div>

<p>Paragraphe 1.</p>
<p>Paragraphe 2.</p>
<p>Paragraphe 3.</p>

La disposition en tableaux

Les tableaux HTML sont un outil idéal pour afficher des données tabulaires. Mais aux débuts du Web, alors que même les propriétés CSS de base n'étaient pas encore implémentées de façon homogène entre les navigateurs, les développeurs web utilisaient les tableaux afin d'organiser le contenu de pages entières et plaçaient les titres, bas de page, etc. dans des lignes et colonnes d'un grand tableau. Cette méthode de contournement faisait l'affaire à l'époque mais n'était pas sans inconvénient : peu de flexibilité, une lourdeur pour la structure du HTML, des difficultés à déboguer et surtout, une sémantique dévoyée (les utilisateurs de lecteur d'écran ne peuvent pas naviguer naturellement avec cette structure).

La façon dont un tableau s'affiche ainsi sur une page web est liée aux propriétés CSS pour les éléments des tableaux. On peut toutefois utiliser ces propriétés pour disposer les éléments qui ne sont pas des tableaux.

Dans l'exemple qui suit, on utilise cette technique. Cette dernière est une méthode principalement historique à utiliser pour les vieux navigateurs qui ne prennent pas en charge les boîtes flexibles ou les grilles.

Prenons un exemple avec un formulaire HTML. Chaque élément de saisie possède un libellé. On a également ajouté un paragraphe avec une légende. Chaque couple libellé/champ est englobé dans un élément <div> qu'on utilisera pour la disposition.

<form>
  <p>Pour commencer, veuillez nous indiquer votre nom et votre âge.</p>
  <div>
    <label for="fname">Prénom:</label>
    <input type="text" id="fname">
  </div>
  <div>
    <label for="lname">Nom de famille:</label>
    <input type="text" id="lname">
  </div>
  <div>
    <label for="age">Âge:</label>
    <input type="text" id="age">
  </div>
</form>

Voyons ensuite le CSS utilisé. La plupart du CSS est plutôt classique sauf pour la propriété display. On a indiqué aux éléments <form>, <div>, <label> et <input> de se comporter comme des tableaux, lignes et cellules. Autrement dit, leur organisation visuelle sera celle d'un tableau HTML et les libellés et les champs seront ainsi alignés. Il ne reste plus qu'à ajouter des dimensions, des marges, etc. afin de finaliser l'ensemble.

Vous pouvez voir que le paragraphe avec la légende est ciblé par display: table-caption; afin qu'il se comporte comme la légende d'un tableau (voir l'élément <caption>) et par caption-side: bottom; afin que la légende soit affichée en bas du tableau (bien que le paragraphe apparaisse avant les éléments <input> dans le code source).

html {
  font-family: sans-serif;
}

form {
  display: table;
  margin: 0 auto;
}

form div {
  display: table-row;
}

form label, form input {
  display: table-cell;
  margin-bottom: 10px;
}

form label {
  width: 200px;
  padding-right: 5%;
  text-align: right;
}

form input {
  width: 300px;
}

form p {
  display: table-caption;
  caption-side: bottom;
  width: 300px;
  color: #999;
  font-style: italic;
}

La disposition en colonnes

Le module de spécification CSS pour la disposition en colonnes nous permet d'organiser le contenu d'une page sur plusieurs colonnes, à la façon du texte dans un journal. Bien que cela ne s'applique pas à tous les contextes web (on ne veut pas forcer un utiliser à défiler de haut en bas puis de bas en haut), organiser le contenu en colonnes peut s'avérer utile selon les cas.

Pour transformer un bloc en conteneur multi-colonnes, on utilise la propriété column-count qui indique le nombre de colonnes à organiser ou la propriété column-width qui indique au navigateur de remplir le conteneur avec le plus de colonnes de cette largeur.

Dans l'exemple qui suit, on a un fragment HTML qui contient un élément <div> avec la classe container.

<div class="container">
    <h1>Disposition en colonnes</h1>

    <p>Paragraphe 1.</p>
    <p>Paragraphe 2.</p>

</div>

On utilise la propriété column-width avec 200 pixels sur ce conteneur. Le navigateur crée alors autant que possible de colonnes de 200 pixels puis partage l'espace restant entre les colonnes.
