Chaque propriété CSS possède un type de valeur qui définit l'ensemble de valeurs qui peut être utilisé avec cette propriété. La page MDN d'une propriété CSS vous indiquera les types de valeurs utilisables pour une propriété donnée. Dans cet article, nous allons aborder les types de valeur les plus courants et les valeurs et unités associées.

Prérequis : Savoir utiliser un ordinateur, avoir installé les logiciels de base pour le développement web, avoir une connaissance de base pour la manipulation des fichiers, connaître les bases de HTML (voir Une introduction à HTML), et avoir une idée du fonctionnement de CSS (voir Les premiers pas en CSS.)
Objectif : Apprendre les différents types de valeurs et unités utilisés dans les propriétés CSS.

Qu'est-ce qu'une valeur CSS ?

Dans les spécifications de CSS ou sur les pages de MDN décrivant les propriétés, vous pourrez identifier les types de valeur car ils sont entourés de chevrons. Par exemple, on pourra trouver <color> ou <length>. Lorsque vous verrez le type de valeur <color> indiqué comme valide pour une propriété donnée, cela signifie que n'importe quelle couleur valide peut être utilisée comme valeur pour cette propriété (les couleurs valides sont listées sur la page de référence <color>).

Note : Vous pourrez aussi voir mentionné des types de donnée. C'est une expression synonyme : un type de donnée CSS ou un type de valeur CSS représentent la même chose. Le terme « valeur » fait référence à toute expression valide pour un type de valeur.

Note : On utilise des chevrons afin de différencier les types de valeur CSS des propriétés (par exemple, on a la propriété color et le type de valeur <color>). On peut se dire qu'on risque de confondre ces types avec des éléments HTML mais le contexte est généralement très différent : il y a peu de risque de confusion.

Dans l'exemple suivant, on définit la couleur d'un titre avec un mot-clé tandis que l'arrière-plan est défini avec une fonction rgb() :

h1 {
  color: black;
  background-color: rgb(197,93,161);
}

Un type de valeur CSS est une façon de définir un ensemble de valeurs valides. Ainsi, si on voit que <color> est valide, on n'a pas à se demander quels types de couleurs peuvent être utilisés entre les mots-clés, les valeurs hexadécimales, les fonctions rgb(), etc. On pourra dès lors utiliser n'importe quelle couleur disponible tant que celle-ci est prise en charge par le navigateur. La page de référence de MDN pour chaque valeur vous fournira les indications de compatibilité. Ainsi, si vous consultez <color>, vous pouvez vous reporter à la section « Compatibilité des navigateurs » pour en savoir plus sur la prise en charge.

Voyons maintenant certains des types de valeurs et unités fréquemment utilisés. Des exemples seront présents afin que vous puissiez essayer différentes valeurs.

Nombres, longueurs et pourcentages

Il existe différents types de valeur numériques en CSS :

Type de données Description
<integer> Une valeur de type <integer> est un nombre entier (par exemple 1024 ou -55).
<number> Une valeur de type <number> représente un nombre décimal, qui peut (ou non) avoir une partie flottante. Les valeurs 0.255, 128, ou -1.2 sont des valeurs de ce type.
<dimension> Une valeur de type <dimension> est une valeur de type <number> associée à une unité. Ainsi, 45deg, 5s, ou 10px sont des dimensions. <dimension> est une catégorie « chapeau » qui regroupe les types <length>, <angle>, <time>, et <resolution>.
<percentage> Une valeur de type <percentage> représente un pourcentage, c'est-à-dire une fraction relative à une autre valeur. 50% est un exemple d'une telle valeur. Les pourcentages sont toujours relatifs à une autre quantité. Ainsi, si la longueur d'un élément est exprimée en pourcentage, celle-ci sera relative à la longueur de l'élément parent.

Longueurs

Le type numérique que vous verrez le plus souvent est le type <length> qui permet de représenter des longueurs. 10px ou 30em sont des valeurs de longueur. Il existe deux types de longueurs en CSS : les longueurs relatives et les longueurs absolues. La distinction entre les deux est importante à la compréhension de la taille des éléments.

Unités de longueurs absolues

Les unités qui suivent sont des unités de longueurs absolues. Cela signifie qu'elles ne sont pas relatives à autres choses et qu'elles correspondent toujours à la même taille quel que soit le contexte.

Unité Nom Équivalence
cm Centimètres 1cm = 38px = 25/64in
mm Millimètres 1mm = 1/10e de 1cm
Q Quart de millimètres 1Q = 1/40e de 1cm
in Pouces (inches) 1in = 2.54cm = 96px
pc Picas 1pc = 1/6e de 1in
pt Points 1pt = 1/72e de 1in
px Pixels 1px = 1/96e de 1in

La plupart de ces unités sont notamment utiles lors de l'impression plutôt que pour l'affichage sur un écran. On utilise par exemple peu l'unité cm (centimètres) pour les écrans. La seule unité qu'on rencontrera dans ce contexte reste px (pixels).

Unités de longueurs relatives

Les unités de longueurs relatives sont relatives à une autre valeur comme la taille de l'élément parent, la dimension de la zone d'affichage, etc. L'avantage des unités relatives est qu'elles permettent d'avoir une disposition homogène quelle que soit l'échelle d'affichage. Certaines des unités les plus utiles en CSS sont listées dans le tableau qui suit.

Unité Relative à
em La taille de la police de l'élément parent pour les propriétés typographiques telles que font-size ou la taille de la police de l'élément pour les autres propriétés comme width.
ex La hauteur d'un « x » dans la police de l'élément.
ch La chasse du glyphe « 0 » dans la police de l'élément.
rem La taille de la police de l'élément racine.
lh La hauteur de la ligne de l'élément.
vw 1% de la largeur de la zone d'affichage (viewport).
vh 1% de la hauteur de la zone d'affichage (viewport).
vmin 1% de la plus petite des dimensions de la zone d'affichage (viewport).
vmax 1% de la plus grande des dimensions de la zone d'affichage (viewport).

La largeur de la deuxième boîte est exprimée en vw. em et rem

em et rem sont deux unités de longueurs relatives que vous rencontrerez fréquemment pour le dimensionnement des boîtes ou du texte. Penchons nous un peu plus sur celles-ci et leurs différences. Cela sera utile pour certains sujets plus complexes comme la mise en forme du texte ou les dispositions CSS.

Dans l'exemple qui suit, on a un code HTML qui décrit un ensemble de listes imbriquées : trois listes sont présentes. Ce code HTML est utilisé deux fois. La seule différence entre ces deux blocs est l'utilisation de l'unité em dans le premier et l'utilisation de l'unité rem dans le second.

Pour commencer, on définit la taille de la police à 16px sur l'élément <html>.

L'unité em indique « la taille de la police de mon élément parent » pour les propriétés liées à la typographie. Les éléments <li> à l'intérieur des éléments <ul> dont la classe est ems seront dimensionnés en fonction de leur parent. Ainsi, pour chaque niveau d'imbrication, la taille augmente progressivement car chacun est dimensionné avec 1.3em et prend donc 1.3 fois la taille de la police de son parent.

L'unité rem indique « la taille de la police de l'élément racine » (rem signfiant « root em » en anglais). Les éléments <li> à l'intérieur de l'élément <ul> dont la classe est rems seront dimensionnés en fonction de l'élément racine (<html>). Cela signifie que, quel que soit le niveau d'imbrication, le contenu ne s'agrandira pas.

Toutefois, si on modifie la propriété font-size de l'élément <html> dans le code CSS, on verra que tous les éléments sont modifiés relativement (aussi bien ceux dimensionnés avec rem que ceux dimensionnés avec em).

Pourcentages

Dans de nombreux cas, un pourcentage sera traité comme une longueur. Un pourcentage est toujours relatif à une autre valeur. Ainsi, si on définit la propriété font-size d'un élément, ce sera un pourcentage de la valeur de la propriété font-size de l'élément parent. Si on utilise un pourcentage pour une valeur de largeur (width)n celle-ci sera proportionnelle à la largeur de l'élément parent.

Dans l'exemple qui suit, les deux boîtes dimensionnées avec des pourcentages et les deux boîtes dimensionnées avec des pixels ont les mêmes noms de classe. Les deux ensembles sont respectivement dimensionnés avec une largeur de 40% et 200px.

La différence apportée au second ensemble de deux boîtes est qu'il est contenu dans un conteneur de 400 pixels de large. La boîte large de 200px est aussi large que celle du premier groupe. En revanche, celle dont la largeur vaut 40% sera aussi large que 40% de 400px, ce qui est beaucoup plus étroit que pour le premier groupe !

Essayez de modifier la largeur du conteneur du second groupe ou les valeurs de pourcentage pour observer les impacts.

On notera que, bien que certaines valeurs acceptent une longueur ou un pourcentage, certaines valeurs acceptent uniquement des longueurs. Si les valeurs autorisées incluent le type <length-percentage>, cela signifie qu'on peut utiliser une longueur ou un pourcentage. Si la valeur autorisée n'inclut que <length>, on ne pourra pas utiliser de pourcentage.

Nombres

Certains types de valeurs fonctionnent avec des nombres, sans unité. On a par exemple la propriété opacity qui contrôle l'opacité d'un élément. Cette propriété permet d'utiliser une valeur comprise entre 0 (complètement transparent) et 1 (complètement opaque).

Couleur

Il existe de nombreuses méthodes pour définir une couleur en CSS dont certaines sont apparues (et ont été implémentées) plus récemment que d'autres. Les valeurs de couleur peuvent être utilisées pour la couleur du texte, celle de l'arrière-plan ou autre.

Le système de couleurs disponible sur les ordinateurs modernes est exprimé sur 24 bits ce qui permet d'afficher environ 16.7 millions de couleurs différentes en combinant différentes intensités de rouge, vert, bleu dont chacune peut prendre 256 valeurs (256 x 256 x 256 = 16 777 216). Voyons certaines méthodes pour utiliser les couleurs en CSS.

Note : Dans ce tutoriel, nous verrons certaines méthodes de définition de couleur qui sont largement implémentées par les différents navigateurs. Il en existe d'autres mais elles ne sont pas aussi bien prises en charge ni aussi fréquemment utilisées.

Mots-clés pour les couleurs

La plupart du temps, dans les exemples présents sur MDN, vous verrez des mots-clés utilisés pour définir des couleurs. Ceux-ci permettent une définition simple et compréhensible d'une couleur. Il existe une large palette de tels mots-clés. Vous pouvez consulter la liste sur la page du type <color>.

Essayez de modifier les valeurs des couleurs dans les exemples qui suivent afin de mieux comprendre comment elles fonctionnent.

Valeurs RGB hexadécimales

Le deuxième type de valeur utilisé pour les couleurs sont sans doute les codes hexadécimaux. Chaque valeur hexadécimale se compose d'un dièse (#) suivi de six chiffres hexadécimaux (chacun peut prendre une valeur entre 0 et f (which qui représente 15 ; un chiffre hexadécimal est donc l'un des chiffres parmi 0123456789abcdef). Les trois paires de ces chiffres correspondent chacune à un des canaux : rouge, vert puis bleu. On peut ainsi indiquer la valeur de la composante parmi les 256 valeurs disponibles (16 x 16 = 256.)

Ces valeurs sont un peu plus complexes et moins faciles à comprendre mais elles sont beaucoup plus flexibles que les mots-clés et on peut utiliser les valeurs hexadécimales pour représenter n'importe quelle couleur.

Valeurs RGB et RGBA

La troisième notation que nous allons voir est RGB. Une valeur RGB est donnée par une fonction — rgb() — dont les trois paramètres représentent respectivement la composante rouge (red), verte (green) et bleue (blue). Contrairement aux valeurs hexadécimales, chaque canal n'est pas représenté par deux chiffres hexadécimaux mais par un nombre en base 10 compris entre 0 et 255 ; ce qui peut rendre la compréhension plus simple.

Revoyons notre dernier exemple afin d'utiliser des couleurs RGB :

{{EmbedGHLiveSample("css-examples/learn/values-units/color-rgb.html", '100%', 700)}}

On peut également utiliser des couleurs RGBA. Celles-ci fonctionnent comme les couleurs RGB (on peut donc utiliser ici n'importe quelle valeur RGB) mais utilisent une quatrième valeur qui représente le canal alpha de la couleur (qui définit l'opacité). Si on utilise 0 comme valeur pour cette composante, la couleur sera complètement transparente tandis qu'une valeur à 1 indiquera une couleur totalement opaque. Les valeurs intermédiaires fournissent différents niveaux d'opacité.

Note : La définition du canal alpha d'une couleur est différente de l'utilisation de la propriété opacity vue plus haut. Lorsqu'on utilise opacity, l'élément et tout ce qu'il contient devient opaque ; avec les couleurs RGBA, seule la couleur est modifiée.

Dans cet exemple, n'hésitez pas à modifier les valeurs pour les canaux alpha afin de voir comment les couleurs sont modifiées.

Note : Les navigateurs récents ont réglé leurs implémentations afin que rgba() et rgb(), comme hsl() et hsla() (voir ci-après), soient des synonymes l'un de l'autre et se comportent de la même façon. Ainsi rgba() et rgb() acceptent des valeurs de couleurs avec ou sans valeur pour le canal alpha. Vous pouvez modifier l'exemple qui précède en utilisant des fonctions rgb() à la place de rgba() et voir que les couleurs fonctionnent toujours. Toutefois, utiliser les deux fonctions distinctement permet d'indiquer plus explicitement si une couleur transparente est utilisée (et également, dans une moindre mesure, de bénéficier d'une meilleure compatibilité des navigateurs).

Valeurs HSL et HSLA

Bien que légèrement moins bien prises en charge que RGB, le modèle de couleur HSL a été implémenté dans les navigateurs afin de correspondre aux façons de définir les couleurs utilisées par les designers. Plutôt que d'utiliser des composantes rouge, verte, bleue, une fonction hsl() accepte des valeurs de teinte (hue en anglais), saturation et de luminosité (lightness en anglais) pour créer une couleur parmi les 16,7 millions possibles différemment :

  • Teinte : la nuance de base pour la couleur. Cette valeur est comprise entre 0 et 360 et représente un angle sur la roue chromatique.
  • Saturation : le taux de saturation de la couleur. Une valeur comprise entre 0% et 100%. 0% indique aucune couleur (ce sera alors un niveau de gris) tandis que 100% indique une saturation totale.
  • Luminosité : la clarté de la couleur. Une valeur comprise entre 0% et 100% où 0% indique aucune luminosité (noir) et où 100% correspond à une luminosité maximale (blanc).

Position

Le type de valeur <position> représente un ensemble de coordonnées sur un espace en deux dimensions. Les valeurs de ce type sont utilisées pour positionner un élément sur une page comme une image d'arrière-plan (grâce à la propriété background-position). Les valeurs pour ce type peuvent être des mots-clés comme top, left, bottom, right, et center qui permettent d'aligner les objets sur l'un des côtés (respectivement haut, gauche, bas, droit) d'une boîte en deux dimensions ou des longueurs qui représenteront alors des décalages par rapport aux côtés haut ou gauche de la boîte.

Une position consiste en deux valeurs : la première correspondant à l'axe vertical et la seconde à l'axe vertical. Si une seule valeur est fournie, l'autre prendra la valeur center par défaut.

Chaînes de caractères et identifiants

Dans les exemples qui précèdent, vous avez pu voir certains mots-clés utilisés comme valeurs (par exemple pour le type <color>, on avait des mots-clés tels que red, black, rebeccapurple, et goldenrod). Ces mots-clés sont plus précisément des identifiants, c'est-à-dire des valeurs spéciales, comprises par CSS. Ces valeurs ne sont pas entourées de doubles quotes (") car elles ne sont pas traitées comme des chaînes de caractères.

Il existe des endroits où utiliser des chaînes de caractères en CSS, par exemple lorsqu'on définit du contenu généré. Dans ce cas, la valeur utilisée est mise entre doubles quotes afin d'indiquer que c'est une chaîne de caractères. Dans l'exemple qui suit, on utilise un identifiant pour un mot-clé d'une couleur ainsi qu'une chaîne de caractères pour du texte généré.

Fonctions

Le dernier type de valeur que nous verrons ici est le groupe de valeurs appelé « fonctions ». En programmation, une fonction est une portion de code réutilisable qui peut être exécutée plusieurs fois afin de réaliser une tâche répétitive sans multiplier les efforts et du développeur et de l'ordinateur. Les fonctions sont généralement associées à des langages tels que JavaScript, Python, ou C++, mais elles existent également en CSS comme valeurs pour les propriétés. Nous avons déjà vu ici certaines fonctions avec les couleurs : rgb(), hsl(), etc. La valeur utilisée pour renvoyer une image depuis un fichier, url(), est également une fonction.

Une valeur plus proche des fonctions « traditionnelles » est la fonction CSS calc(). Cette fonction permet de réaliser certains calculs en CSS. Elle s'avère particulièrement utile lorsqu'on souhaite manipuler des valeurs qui ne sont pas connues au moment de l'écriture du CSS et qui doivent être calculées à l'exécution, par le navigateur.

Dans l'exemple qui suit, on utilise calc() pour définir la largeur de la boîte à 20% + 100px. La composante de 20% est calculée relativement à la largeur du conteneur parent .wrapper et changera selon cette largeur. On ne peut pas réaliser ce calcul au préalable car on ne sait pas ce que les 20% représenteront. Aussi, on utilise calc() pour demander au navigateur de faire ce calcul.

Résumé

Voici une rapide introduction aux types de valeurs et aux unités les plus communément utilisés. Pour la liste exhaustive, vous pouvez consulter la page de référence Valeurs et unités CSS ; vous utiliserez souvent ces notions au cours de ces modules.

La principale notion à retenir est que chaque propriété accepte un ensemble de types de valeurs défini et que chaque type de valeur est défini pour expliquer les valeurs qu'il contient.

Ainsi, vous pouvez désormais comprendre que le type <image> permet également de créer des dégradés de couleurs. C'est un aspect utile mais pas nécessairement évident de prime abord.
