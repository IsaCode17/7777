Cet article explique comment configurer un serveur de test local simple sur votre machine, et les bases pour l'utiliser.

Prérequis : Vous devez au préalable savoir comment Internet fonctionne et ce qu'est un serveur Web.
Objectif: Vous apprendrez à configurer un serveur de test local.

Fichiers locaux et fichiers distants

Dans les différents articles de cette section, nous vous demandons d'ouvrir les exemples directement dans le navigateur — vous pouvez le faire en double cliquant sur le fichier HTML, en déposant celui-ci dans la fenêtre de votre navigateur, ou en faisant Fichier > Ouvrir… et naviguer jusqu'au fichier HTML, etc. Il y a beaucoup de manières d'y arriver.

Lorsque l'adresse web visible dans votre navigateur commence par file:// suivi du chemin d'accès dans votre système de fichiers, cela indique que c'est un fichier local qui est chargé. Par contre, si vous consultez un de nos exemples hébergés sur GitHub (ou n'importe quel autre serveur distant), l'adresse web commencera par http:// ou https:// ; dans ce cas le fichier a été servi via HTTP.

Le problème des tests avec des fichiers locaux

Certains exemples ne fonctionneront pas en local. Il y a plusieurs raisons possibles, dont les plus courantes sont :

  • Ils sont basés sur des requêtes asynchrones. Certains navigateurs comme Chrome n'exécutent pas de requêtes asynchrones (voyez Récolter des données depuis le serveur ) si vous lancez simplement l'exemple comme un fichier local. Cela est dû à des restrictions de sécurité (voir Sécurité des sites web pour en savoir plus).
  • Ils mettent en œuvre un langage spécifique, exécuté côté serveur. De tels langages « serveur » (comme PHP ou Python) nécessitent un environnement spécifique, fourni par le serveur pour interpréter le code et donner des résultats.

Créer un serveur HTTP local simple

Pour contourner le problème des requêtes asynchrones, nous devons tester de tels exemples en les exécutant depuis un serveur web local. C'est-à-dire que nous allons utiliser notre propre ordinateur pour servir lui-même le contenu web au navigateur. Le module http.server de Python permet une mise en œuvre simple de cette solution.

Voilà la marche à suivre :

  1. Installez Python. Si vous utilisez GNU/Linux ou macOS, un environnement Python est sans doute déjà disponible sur votre machine. Si vous utiliser Windows vous pourrez trouver un installeur depuis la page d'accueil de Python (on y trouve toutes les instructions) :

    1. Allez à python.org
    2. Sous Télécharger, cliquez le lien pour Python "3.xxx".
    3. Tout en bas de la page, téléchargez le fichier pointé par le lien Windows x86 executable installer.
    4. Exécutez ce programme quand le téléchargement est fini.
    5. Sur la première page de l'installeur, assurez-vous d'avoir coché la case "Ajouter Python 3.xxx to PATH".
    6. Cliquez Installer, puis Fermer quand l'installation est complète.

  2. Ouvrez votre invite de commandes (Windows) ou votre terminal (OS X et GNU/Linux). Pour vérifier que l'installation précédente s'est déroulée correctement, entrez la commande suivante :

    python -V

  3. Elle devrait retourner un numéro de version. Si c'est le cas, en utilisant la commande cd, déplacez vous dans votre répertoire de travail qui contient l'exemple.

    #inclure le nom du dossier pour y s'y rendre,
#par exemple
cd Bureau
# utiliser deux points pour remonter dans
#le dossier parent si nécessaire
cd ..

  4. Entrez la commande pour démarrer le serveur dans ce dossier.

    python -m http.server

  5. Par défaut, le serveur que vous venez de démarrer affiche la liste des fichiers du dossier sur un serveur de développement, sur le port 8000. Vous pouvez accéder à ce serveur en saisissant l'URL localhost:8000 dans votre navigateur web. Vous verrez le listing du dossier dans lequel le serveur tourne — cliquez le fichier HTML que vous voulez exécuter.

Note : Si le port 8000 est occupé, vous pouvez choisir un autre port en spécifiant une autre valeur après la commande par exemple python -m http.server 7800. Vous pouvez maintenant accéder à votre contenu à l'adresse localhost:7800.

Exécuter localement des programmes côté serveur

Le module http.server de Python est utile, mais il ne sait pas comment exécuter du code écrit dans des langages tels que PHP, Python ou JavaScript. Pour cela, vous aurez besoin de quelque chose en plus qui dépend du language serveur que vous souhaiterez exécuter. Voici quelques exemples :

  • Pour exécuter du code Python coté-serveur, vous aurez besoin d'utiliser un framework web Python. Vous pouvez découvrir comment utiliser le framework Django en lisant Django Web Framework (Python). Flask est une alternative à Django, un peu plus légère. Pour l'exécuter, vous aurez besoin d'installer Python/PIP, puis Flask en utilisant pip3 install flask. À ce point, vous devriez être capable d'exécuter les exemples Python Flask en utilisant par exemple python3 python-example.py, puis consulter localhost:5000 dans votre navigateur.
  • Pour exécuter du code Node.js (JavaScript) côté serveur, vous aurez besoin d'utiliser Node.js tel quel ou un framework construit par dessus ce dernier. Express est un bon choix — voir Express Web Framework (Node.js/JavaScript).
  • Pour exécuter du code serveur PHP, après avoir installé l'environnement PHP, vous pouvez lancer le serveur de développement PHP : 
    $ cd chemin/vers/le/code/php
$ php -S localhost:8000
