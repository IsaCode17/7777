Baik Anda seorang ahli atau baru memulai, artikel yang dapat Anda temukan mulai dari halaman ini akan membantu Anda saat Anda mengerjakan pengembangan Mozilla.

Topik dokumentasi

Getting Started
Panduan pemula langkah demi langkah untuk terlibat dengan Mozilla.
Working with Mozilla Source Code
Ringkasan kode, cara mendapatkan kode, dan panduan gaya pengkodean.
Build Instructions
Cara membangun Firefox, Thunderbird, SeaMonkey, atau aplikasi Mozilla lainnya.
Development process overview
Gambaran umum dari seluruh proses pengembangan Mozilla.
Managing multiple profiles
Saat bekerja dengan Firefox versi prarilis, memiliki beberapa profil Firefox, seperti satu profil untuk setiap saluran, atau untuk berbagai jenis pengujian, akan sangat membantu.
Automated Testing
Cara menjalankan pengujian otomatis Mozilla, dan cara menulis pengujian baru.
How to submit a patch
Setelah tambalan Anda ditulis, Anda perlu memeriksanya ke pohon. Artikel ini menjelaskan proses peninjauan dan cara mendapatkan persetujuan tambalan Anda.
Getting documentation updated
Bagaimana memastikan bahwa dokumentasi selalu diperbarui saat Anda mengembangkan.
Mozilla modules and module ownership
Artikel ini memberikan informasi tentang modul Mozilla, apa peran pemilik modul, dan bagaimana pemilik modul dipilih.
Code snippets
Contoh kode yang berguna untuk berbagai hal yang mungkin perlu Anda ketahui cara melakukannya.
Mozilla development strategies
Kiat tentang cara memanfaatkan waktu Anda untuk mengerjakan proyek Mozilla
Debugging
Temukan tip dan panduan berguna untuk men-debug kode Mozilla.
Performance
Panduan kinerja dan utilitas untuk membantu Anda membuat kode Anda bekerja dengan baik (dan bermain baik dengan orang lain).
The Mozilla platform
Informasi tentang cara kerja platform Mozilla.
Adding APIs to the navigator object {{ gecko_minversion_inline("9.0") }}
Cara menambah file {{ domxref("window.navigator") }} objek dengan API tambahan.
Interface Compatibility
Panduan untuk memodifikasi API skrip dan biner di Mozilla.
Customizing Firefox
Informasi tentang membuat versi khusus Firefox.
Virtual ARM Linux environment
Cara menyiapkan emulator ARM yang menjalankan Linux untuk menguji kode khusus ARM, tetapi tidak harus khusus platform. Berguna untuk pengembang seluler.
Obsolete Build Caveats and Tips
Tempat untuk meletakkan tips build yang tidak lagi relevan untuk membuat kode versi terbaru dari main tetapi relevan saat membuat basis kode lama.

Alat

Bugzilla
Database Bugzilla yang digunakan untuk melacak masalah untuk proyek Mozilla.
MXR
Jelajahi dan cari repositori kode sumber Mozilla di Web.
Bonsai
Alat Bonsai memungkinkan Anda mengetahui siapa yang mengubah file apa di repositori, dan kapan mereka melakukannya.
Mercurial
Sistem kontrol versi terdistribusi yang digunakan untuk mengelola kode sumber Mozilla.
Tinderbox
Tinderbox menunjukkan status pohon (apakah saat ini berhasil dibangun atau tidak). Periksa ini sebelum check in dan out, untuk memastikan Anda bekerja dengan pohon yang berfungsi.
Crash tracking
Informasi tentang Socorro and Talkback sistem pelaporan kecelakaan.
Performance tracking
Lihat informasi kinerja untuk proyek Mozilla.
Callgraph
Alat untuk membantu melakukan analisis statis kode Mozilla dengan membuat grafik panggilan secara otomatis.
Developer forums
Daftar forum diskusi khusus topik tempat Anda dapat berbicara tentang masalah pengembangan Mozilla.
Mozilla Platform Development Cheat Sheet
Daftar Brian Bondy tentang informasi yang sering dirujuk untuk pengembang platform.
