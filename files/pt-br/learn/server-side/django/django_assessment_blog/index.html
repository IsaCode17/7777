Nessa avaliação você vai usar o conhecimento de Django que adquiriu até agora no módulo Django Web Framework (Python) para criar um blog bem simples.

Pré-Requisitos: Antes de tentar fazer esta avaliação você deve ter passado por todos os outros artigos deste módulo.
Objetivo: Testar a compreensão dos fundamentos do Django, incluindo configurações de URL, modelos, views, formulários e templates.

Resumo do projeto

As páginas que precisam ser mostradas, suas URLs e outros requisitos estão abaixo:

Página URL Requisitos
Página inicial / e /blog/ Uma página inicial descrevendo o site.
Lista de todos os artigos do blog /blog/blogs/

Lista de todos os artigos do blog:

  • Acessível para todos os usuários através de um link na barra lateral.
  • Lista artigo pela data (de forma decrescente).
  • Lista paginada em grupos de 5 artigos.
  • Os itens da lista mostram o título do blog, a data do artigo e o autor.
  • O nome dos artigos são linkados com a página de detalhes do blog.
  • Os bloggers (nome dos autores) são linkados com a página de detalhes dos autores do blog.
Página de detalhes do autor do blog (o blogger) /blog/blogger/<author-id>

Informação de um autor específico (pelo id) e a lista dos artigos do seu blog:

  • Acessível para todos os usuários pelos links do autor nos artigos do blog etc.
  • Contém algumas informações biográficas do blogger/autor.
  • Lista artigo pela data (de forma decrescente).
  • Não paginado.
  • Os itens da lista mostram somente o nome do artigo do blog e a data.
  • O nome dos artigos do blog são linkados com a página de detalhes do blog.
Página de detalhes do artigo do blog /blog/<blog-id>

Detalhes do artigo.

  • Acessível para todos os usuários pela lista de artigos do blog.
  • Contém na página do artigo: nome, autor, data e conteúdo.
  • Comentários do artigo devem ser mostrados no final da página.
  • Comentários devem ser ordenados de forma crescente (do mais antigo para o mais novo).
  • Contém link para adicionar comentários no final da página para usuários logados (veja a página do formulário de comentário)
  • Artigos e comentários precisam exibir somente texto simples. Não é necessário suporte de nenhum tipo de marcação HTML (como links, imagens, negrito/itálico, etc).
Lista de todos os bloggers /blog/bloggers/

Lista dos bloggers no sistema:

  • Acessível para todos os usuários pela barra lateral do site.
  • O nome dos bloggers são linkados com as página de detalhes do autor do blog.
Página do formulário de comentário /blog/<blog-id>/create

Cria comentário para o artigo do blog:

  • Acessível somente por usuários logados, através do link no final da página de detalhes do artigo.
  • Mostra formulário com descrição para novos comentários (a data do artigo e do blog não são editáveis).
  • Depois de um comentário ter sido postado, a página será redirecionada de volta para a respectiva página do artigo.
  • Usuários não podem editar ou deletar seus comentários.
  • Usuários não logados serão direcionados para a página de login para logar, antes de adicionarem o comentário. Depois de logados, eles serão redirecionados de volta para a página do artigo que tentaram criar o comentário.
  • Página de comentários deve incluir o nome/link do artigo que está recebendo o comentário.
Páginas de autenticação do usuário /accounts/<standard urls>

Páginas de autenticação padrão do Django para fazer login, logout e configurar senhas:

  • Login/logout deve ser acessível pelos links da barra lateral.
Site do admin /admin/<standard urls>

A página de Admin deve estar ativada para permitir criar/editar/deletar os artigos, autores e comentários do blog (esse é o mecanismo para os bloggers criarem novos artigos para o blog):

  • Os registros do site do Admin devem mostrar a lista de comentários associados (abaixo de cada artigo do blog).
  • Nomes do comentário no site do Admin são criados de forma que a descrição do comentário seja truncada em até 75 caracteres.
  • Outros tipos de registro podem usar uma configuração mais básica.
